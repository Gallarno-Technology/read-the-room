**Multi-badge flex container and severity-aware JS rendering — dashboard shows Passed + Mild language badges simultaneously for tracks with severity >= 1**

## Accomplishments

- Replaced single `<span id="now-playing-badge">` with `<div id="badge-group" class="badge-group">` flex container — foundational multi-badge layout for v1.3
- Added `.badge--mild-language` amber CSS class (distinct alpha from `.badge--evaluating`) and `.badge-group` flex CSS
- Updated `setEvalBadge(evalState, severity)` to append a "Mild language" badge when passed + severity >= 1, remove it otherwise
- Wired severity from both SSE `eval_result` events (`evt.severity || 0`) and `/now-playing` hydration (`data.severity || 0`)
- All 4 web_ui endpoint tests continue to pass

## Decisions Made

- Preserving existing `id="now-playing-badge"` span unchanged inside the new badge-group div avoids any JS constant rename — `const nowPlayingBadge` still works without edits
- `.badge--mild-language` uses slightly lower alpha (0.15 bg / 0.25 border) than `.badge--evaluating` (0.2 / 0.3) and muted text `#c4924a` vs `#d4924a` to distinguish "mild content present but passing" from "actively evaluating"
- `setEvalBadge` purges `.badge--mild-language` before conditionally re-adding — makes the function idempotent on repeated `eval_result` events for the same track
