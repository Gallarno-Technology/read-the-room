## Component Contract: Split Button (`#fsm-split-btn`)

This is the primary new component in Phase 16. It replaces the single `<button id="fsm-toggle">`.

### Structure

```
#fsm-split-btn  (div, position: relative, display: flex, width: 100%, height: 44px)
├── #fsm-toggle  (button, flex: 1, left zone — FSM toggle action)
│     Label: "{Profile Display Name}"  (e.g. "Kids Present")
│     When FSM fresh-install and off: "The Library is Closed"
└── #profile-dropdown-trigger  (button, width: 40px, right zone — dropdown trigger)
      Label: ▾  (U+25BE)
      aria-haspopup="listbox"
      aria-expanded="false" | "true"

#profile-dropdown  (div, position: absolute, top: 100%, left: 0, width: 100%, hidden attr)
├── .profile-option[data-profile="kids_present"]        "Kids Present"
├── .profile-option[data-profile="were_all_adults"]     "We're All Adults"
├── .profile-option[data-profile="above_the_covers"]    "Above The Covers"
└── .profile-option[data-profile="permissive"]          "Permissive"
      Active profile gets ✓ prefix and bold weight
```

### Split Button States

| State | Container class | Left zone text | Left zone bg | Left zone text color | Right zone | Separator |
|-------|----------------|----------------|--------------|----------------------|------------|-----------|
| FSM on | `fsm-split fsm-on` | Profile display name | `--accent` (#c9a84c) | `#0d0b08` | Same gold bg | 1px `--border` line |
| FSM off | `fsm-split fsm-off` | Profile display name | `--surface-raised` (#242018) | `--text` (#e8e0cc) | Same grey bg | 1px `--border` line |
| FSM off, no profile stored | `fsm-split fsm-off` | "The Library is Closed" | `--surface-raised` | `--text` | Same | Same |
| Hover (FSM on) | — | — | `--accent-dim` (#8a6f2e) | `#0d0b08` | Same | Same |
| Hover (FSM off) | — | — | `--surface` (#1c1a14) | `--text` | Same | Same |

Source: CONTEXT.md D-02, D-03, D-04; existing `.fsm-on` / `.fsm-off` CSS classes.

### Dropdown States

| State | Visibility | ✓ position | aria-expanded |
|-------|------------|------------|---------------|
| Closed (default) | `hidden` attribute | N/A | `"false"` |
| Open | no `hidden` attribute | Next to stored active profile | `"true"` |

Dropdown position: `position: absolute; top: 100%; left: 0; width: 100%` relative to `#fsm-split-btn` container (`position: relative`). No JS measurement needed.

Dropdown animation: **instant appear** (no fade) — consistent with existing dashboard interaction patterns (banner, badges all appear instantly). The `fadeIn` keyframe exists only for feed items.

Dropdown z-index: `z-index: 10` — above cards, below nothing (cards are not stacked).

## Interaction Contract

### Click Zone Separation (Critical — CONTEXT.md D-05)

- Clicking `#fsm-toggle` (left zone): toggles FSM state via `POST /fsm` — does NOT change active profile.
- Clicking `#profile-dropdown-trigger` (right zone): toggles dropdown open/close via `e.stopPropagation()` — does NOT trigger FSM toggle.
- Click events on the container `#fsm-split-btn` do NOT have a listener. FSM listener is on `#fsm-toggle` only.

### Dropdown Dismiss Rules (CONTEXT.md D-10)

- Click outside dropdown + outside `#profile-dropdown-trigger`: close dropdown, no profile change.
- Press Escape: close dropdown, no profile change.
- Click on a profile option: save profile via `POST /profile`, update button label, update ✓ indicator, close dropdown. Do NOT change FSM state.
- SSE reconnect event fires: close dropdown (reset to hidden + aria-expanded false).

### Profile Selection Flow

1. User clicks ▾ → dropdown opens (instant)
2. User clicks a profile name → `POST /profile {"profile": "{key}"}` fires immediately
3. On 200 response: update `activeProfile` variable, call `setFsmUI(fsmEnabled, newProfile)`, close dropdown
4. On error: show error in `#fsm-error`, revert visual selection, keep dropdown closed

### API Interactions

| Action | Endpoint | Method | Body | On Success | On Error |
|--------|----------|--------|------|-----------|---------|
| Toggle FSM | `/fsm` | POST | `{"enabled": bool}` | Update `fsmEnabled`, call `setFsmUI` | Revert, show `#fsm-error` for 3s |
| Select profile | `/profile` | POST | `{"profile": "kids_present"}` | Update `activeProfile`, call `setFsmUI` | Revert, show `#fsm-error` for 3s |

Error message format for profile failure: "Could not save profile — try again."
Error message format for FSM failure (unchanged): "Could not update — try again."

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (FSM on) | Profile display name (e.g. "Kids Present") |
| Primary CTA (FSM off) | Profile display name in grey; "The Library is Closed" only on fresh install |
| Dropdown option — Kids Present | "Kids Present" |
| Dropdown option — We're All Adults | "We're All Adults" |
| Dropdown option — Above The Covers | "Above The Covers" |
| Dropdown option — Permissive | "Permissive" |
| Active profile indicator | "✓" (U+2713) prepended to the active option label |
| Profile save error | "Could not save profile — try again." |
| FSM toggle error (unchanged) | "Could not update — try again." |
| Empty state heading (Incident Log) | "No skips yet" (unchanged) |
| Empty state body (Incident Log) | "Skips will appear here in real-time when Family Safe Mode is on." (unchanged) |

No destructive actions in this phase. Profile switching is non-destructive and immediately reversible by selecting a different profile.

Profile display names are the canonical human-readable labels. Key values in state.json and API calls use snake_case (`kids_present`, `were_all_adults`, `above_the_covers`, `permissive`). These two representations must never be mixed in the UI.

Source: CONTEXT.md D-02, D-03, D-04, D-06, D-08; RESEARCH.md `PROFILE_DISPLAY_NAMES` mapping.

## Accessibility Contract

| Element | Requirement |
|---------|------------|
| `#profile-dropdown-trigger` | `aria-haspopup="listbox"`, `aria-expanded` toggled on open/close |
