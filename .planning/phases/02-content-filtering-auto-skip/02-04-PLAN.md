<objective>
Fix LRCLIB error visibility, add a fetch timeout, and ensure lyrics_cache.db pre-exists on the host before docker compose up.

Purpose: LRCLIB connectivity failures are swallowed silently because the except clause does not bind exc. A stalling connection hangs the daemon's executor thread pool indefinitely. Also, if lyrics_cache.db does not exist on the host, the Docker bind-mount creates a directory instead of a file, causing LyricsService to fail on every call with no clear error message. Additionally, content_checker.py's no_lyrics_service fallback path logs at INFO — it should warn so operators notice when the lyrics pipeline is not functioning.
Output: lyrics_service.py with exception binding + timeout; content_checker.py with warning log; Makefile setup target confirmed to pre-create lyrics_cache.db.

Two changes inside `get_lyrics()` in lyrics_service.py. Both are in the LRCLIB fetch block (currently around lines 116-136).

**Change 1: Bind the exception variable and include it in the log.**

Replace:
```python
except (NotFoundError, APIError, Exception):
    log.warning(
        "[LRCLIB] fetch failed for %r by %r", track_name, artist_name
    )
    return LyricsResult(instrumental=False, lyrics=None, cached=False)
```

With:
```python
except (NotFoundError, APIError, Exception) as exc:
    log.warning(
        "[LRCLIB] fetch failed for %r by %r: %s",
        track_name, artist_name, exc,
        exc_info=True,
    )
    return LyricsResult(instrumental=False, lyrics=None, cached=False)
```

The `as exc` binding makes the exception available; `%s exc` adds the message inline; `exc_info=True` adds the full traceback to the log entry so the operator can see the exact error (connection refused, DNS failure, timeout, etc.).

**Change 2: Wrap run_in_executor with asyncio.wait_for (10 second timeout).**

Replace:
```python
search_results = await loop.run_in_executor(
    None,
    lambda: self._api.search_lyrics(
        track_name=track_name,
        artist_name=artist_name,
    ),
)
```

With:
```python
search_results = await asyncio.wait_for(
    loop.run_in_executor(
        None,
        lambda: self._api.search_lyrics(
            track_name=track_name,
            artist_name=artist_name,
        ),
    ),
    timeout=10,
)
```

The `asyncio.wait_for` raises `asyncio.TimeoutError` after 10 seconds if the executor thread has not returned. `asyncio.TimeoutError` is a subclass of `Exception`, so it is caught by the existing `except (NotFoundError, APIError, Exception) as exc:` handler and treated as lyrics unavailable (FILT-05). No additional except clause needed.

Do NOT change anything else in lyrics_service.py — the cache lookup, INSERT OR REPLACE, close(), or _ensure_db() logic is correct.
