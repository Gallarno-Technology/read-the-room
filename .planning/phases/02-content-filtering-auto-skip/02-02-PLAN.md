2. Define SQLite schema constant:
   ```python
   CREATE_TABLE = """
   CREATE TABLE IF NOT EXISTS lyrics_cache (
       spotify_track_id TEXT PRIMARY KEY,
       track_name       TEXT NOT NULL,
       artist_name      TEXT NOT NULL,
       instrumental     INTEGER NOT NULL DEFAULT 0,
       plain_lyrics     TEXT,
       fetched_at       REAL NOT NULL
   );
   CREATE INDEX IF NOT EXISTS idx_fetched_at ON lyrics_cache(fetched_at);
   """
   ```

3. Define `class LyricsService`:
   ```python
   class LyricsService:
       def __init__(self, db_path: str):
           self.db_path = db_path
           self._db: aiosqlite.Connection | None = None
           self._api = LrcLibAPI(user_agent="SpotifyFamilySafe/1.0")
   ```

4. `async def _ensure_db(self)` — lazily open aiosqlite connection and create table:
   ```python
   if self._db is None:
       self._db = await aiosqlite.connect(self.db_path)
       await self._db.executescript(CREATE_TABLE)
   return self._db
   ```

5. `async def get_lyrics(self, track_id, track_name, artist_name) -> LyricsResult`:
   a. Check SQLite cache first:
      ```python
      db = await self._ensure_db()
      async with db.execute(
          "SELECT instrumental, plain_lyrics FROM lyrics_cache WHERE spotify_track_id = ?",
          (track_id,)
      ) as cursor:
          row = await cursor.fetchone()
      ```
   b. If cache hit: log `[CACHE] hit track_id=...`, return `LyricsResult(instrumental=bool(row[0]), lyrics=row[1], cached=True)`
   c. If cache miss: fetch from LRCLIB via `run_in_executor` (lrclibapi is synchronous — uses requests internally):
      ```python
      loop = asyncio.get_event_loop()
      try:
          result = await loop.run_in_executor(
              None, 
              lambda: self._api.get_lyrics(
                  track_name=track_name,
                  artist_name=artist_name,
              )
          )
      except Exception:
          # LRCLIB down or track not found — treat as lyrics unavailable (FILT-05)
          log.warning("[LRCLIB] fetch failed for %r by %r", track_name, artist_name)
          return LyricsResult(instrumental=False, lyrics=None, cached=False)
      ```
   d. Handle LRCLIB response:
      - If `result` is None (404 / not found): return `LyricsResult(instrumental=False, lyrics=None)` — FILT-05
      - If `result.instrumental` is True: cache as instrumental, return `LyricsResult(instrumental=True, lyrics=None)` — FILT-04
      - If `result.plain_lyrics` is not None and not empty string: cache with lyrics, return `LyricsResult(instrumental=False, lyrics=result.plain_lyrics)`
      - If `result.plain_lyrics` is None/empty but `result.synced_lyrics` exists: extract plain text from synced lyrics (strip timestamps like `[00:15.23]` using regex `r'\[\d{2}:\d{2}\.\d{2,3}\]\s*'`), cache, return result
      - If both are None/empty and instrumental is False: treat as lyrics unavailable (FILT-05) — cache this result too to avoid re-fetching
   e. Cache every result in SQLite:
      ```python
      await db.execute(
          """INSERT OR REPLACE INTO lyrics_cache
             (spotify_track_id, track_name, artist_name, instrumental, plain_lyrics, fetched_at)
             VALUES (?, ?, ?, ?, ?, ?)""",
          (track_id, track_name, artist_name, int(instrumental), plain_lyrics, time.time())
      )
      await db.commit()
      ```
   f. Log: `[LRCLIB] fetched track=%r artist=%r instrumental=%s has_lyrics=%s`

6. `async def close(self)`:
   ```python
   if self._db is not None:
       await self._db.close()
       self._db = None
   ```

7. All LRCLIB calls MUST be in `run_in_executor` — the library uses synchronous `requests` internally (Pitfall 3 from research).

8. Handle `lrclibapi` 404 behavior: wrap in try/except. The library may raise an exception on 404 (Open Question 2 from research). Catch any exception and treat as lyrics unavailable.
