**Change 1: Fix save_state() to read-merge before writing.**

Replace the current save_state() function (lines 68-76) with a version that reads the existing disk state, merges the daemon's fields on top, then writes back. This ensures keys written externally (like family_safe_mode from the Makefile) are never dropped.

New save_state() implementation:
```python
def save_state(daemon_fields: dict) -> None:
    """Merge daemon_fields into the current on-disk state and write back.

    Reads the existing state.json first so that keys written externally
    (e.g. family_safe_mode from `make fsm-on`) are not overwritten.

    Direct write (not atomic rename) — os.replace() fails on bind-mounted
    files on Linux (EBUSY). Safe here: the daemon recovers cleanly from a
    missing/corrupt file via load_state().
    """
    on_disk = load_state()
    on_disk.update(daemon_fields)
    with open(STATE_PATH, "w") as f:
        json.dump(on_disk, f)
```

The parameter rename from `state` to `daemon_fields` is intentional — it makes the contract clear. The call site in poll_loop passes `{"last_track_id": track_id}` (only the fields the daemon manages), NOT the full in-memory state dict. Update the call site too (see Change 2).
