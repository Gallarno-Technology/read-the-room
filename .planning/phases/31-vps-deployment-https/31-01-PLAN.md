<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → Caddy (port 443) | All inbound HTTPS traffic; TLS terminates here |
| Caddy → web_ui (8888) | Internal reverse proxy; plain HTTP on Docker network or localhost |
| browser → /auth/callback | OAuth redirect; carries `code` and `state` params |
| .env file → container env | Secrets loaded at compose up; no runtime exposure beyond container |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Spoofing | Let's Encrypt ACME challenge | mitigate | Caddy handles ACME automatically; domain must have A/AAAA record pointing to VPS before first `docker compose up` — operator checklist item in .env.example comments |
| T-31-02 | Tampering | Caddyfile env var injection | accept | `{$APP_DOMAIN}` and `{$WEB_UI_UPSTREAM}` are set by the operator in .env, not from user input; attack surface is local file system only |
| T-31-03 | Repudiation | Cookie without Secure flag sent over HTTP | mitigate | Task 4 enables `secure=True`; browser refuses to send uid cookie over plain HTTP after this change |
| T-31-04 | Information Disclosure | uid cookie readable over HTTP (pre-fix) | mitigate | Resolved by T-31-03 mitigation (secure=True) — cookie no longer transmitted in cleartext |
| T-31-05 | Denial of Service | ACME rate limiting (Let's Encrypt: 5 certs/domain/week) | accept | Rate limit only applies to production domain cert failures; local testing should use APP_DOMAIN=192.168.1.220 with tls internal not applicable here — operators warned to test with a real domain only once |
| T-31-06 | Elevation of Privilege | NETWORK_MODE=bridge misconfiguration breaking Sonos | accept | Sonos is explicitly out of scope on VPS (REQUIREMENTS.md Out of Scope table); local operators keep NETWORK_MODE unset (defaults to host); no privilege escalation vector |
| T-31-07| Information Disclosure | SPOTIFY_REDIRECT_URI hardcoded to LAN IP in .env.example | mitigate | Task 3 updates example to yourdomain.com pattern; LAN IP removed from all uncommented .env.example lines |
</threat_model>
