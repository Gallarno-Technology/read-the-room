---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/JS) |
| Icon library | none (Unicode glyphs only: ×, —, ·, …) |

## Spacing Scale

Declared values (all multiples of 4). Pre-populated from existing `index.html` usage.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding (vertical), inline element gap |
| sm | 8px | Badge internal padding (horizontal), button inner margin, error margin-top |
| md | 16px | Card padding, element spacing within card |
| lg | 24px | Card margin-bottom, section gaps |
| xl | 32px | Page horizontal padding |
| 2xl | 48px | (available; not used in now-playing card) |
| 3xl | 64px | Page top/bottom padding |

Exceptions:
- Album art: 64px square (matches 3xl token but is a fixed dimension, not spacing)
- Skip button: 44px height (touch target minimum, matches `#fsm-toggle` height)
- SSE dot: 8px square (matches sm token; fixed dimension)
- Card border-radius: 6px (not a spacing token; inherited from `.card`)
- Album art border-radius: 4px (matches badge border-radius; fixed dimension)

## Component Inventory

All implemented inline in `web_ui/templates/index.html`. No separate files.

### Now-Playing Card

```
<div class="card" id="now-playing-card">
  <h2 class="card-heading">Now Playing</h2>
  <hr class="section-divider">
  <!-- idle state (data.status === "idle") -->
  <p id="now-playing-idle" class="empty-body">Nothing playing</p>
  <!-- track state (hidden when idle) -->
  <div id="now-playing-track" style="display:none">
    <div style="display:flex; align-items:center; gap:16px">
      <img id="now-playing-art" src="" alt="" width="64" height="64"
           style="border-radius:4px; flex-shrink:0; display:none">
      <div>
        <div id="now-playing-name" style="font-size:14px; line-height:1.5"></div>
        <div id="now-playing-artist" style="font-size:14px; color:var(--text-dim); line-height:1.5"></div>
        <span id="now-playing-badge" class="badge"></span>
      </div>
    </div>

    <div id="skip-error" style="margin-top:8px; font-size:12px; color:var(--danger); min-height:16px"></div>
  </div>
</div>
```

Placement: inserted between the FSM toggle card and the Incident Log card (CONTEXT.md D-01).

### Album Art (`#now-playing-art`)

- Element: `<img>` with fixed `width="64" height="64"` attribute
- CSS: `border-radius: 4px; flex-shrink: 0`
- Visible: when `album_art_url` is a non-null string
- Hidden: `display:none` when `album_art_url` is null — no placeholder image
- `alt` attribute: set to empty string `""` (decorative; track name/artist already visible in adjacent text)
- Source: Spotify CDN URL from `track_change` event and `GET /now-playing` response

### Eval-State Badge (`#now-playing-badge`)

- Base class: `.badge` (inherits 11px Courier Prime, padding 4px 8px, border-radius 4px)
- Modifier class: one of the six `.badge--*` classes from the Color section above
- State machine (driven by SSE events and hydration):

| Input | Action |
|-------|--------|
| `track_change` event | Reset badge to `.badge--evaluating` + label "Checking…" |
| `eval_result` event (matching `track_id`) | Swap modifier class + update label per D-10 |
| `eval_result` event (mismatched `track_id`) | Ignore silently (NOW-07 guard) |
| `GET /now-playing` response | Set badge to eval_state from response |
| `data.status === "idle"` | Hide badge (card shows idle placeholder) |

Badge label mapping (CONTEXT.md D-10):

| `eval_state` value | Badge label |
|--------------------|-------------|
| `evaluating` | Checking… |
| `passed` | Passed |
| `no-lyrics` | No lyrics |
| `skipped` | Skipped |
| `paused` | Paused |
| `fsm-off` | Monitoring off |

### Skip Button (`#skip-btn`)

- Element: `<button id="skip-btn">`
- CSS: `width:100%; height:44px; margin-top:16px` — matches `#fsm-toggle` sizing
- Font: 13px, weight 600, Courier Prime — matches `#fsm-toggle`
- Default style: `background:var(--surface-raised); color:var(--text); border:1px solid var(--border)` (`.fsm-off` appearance)
- Visible: only when track is playing (`display` managed with card sections)
- Hidden: when `data.status === "idle"` (idle placeholder shown instead of track section)
- Disabled state: `button.disabled = true` on click; re-enabled on fetch settle (SKIP-04)
- Disabled appearance: `opacity:0.5; cursor:not-allowed`

### Skip Error (`#skip-error`)

- Matches `#fsm-error` pattern exactly: `margin-top:8px; font-size:12px; color:var(--danger); min-height:16px`
- Populated on 503 response from `POST /skip`
- Cleared via `setTimeout(3000)` — same pattern as FSM error

## Interaction Contracts

### Page-Load Hydration (NOW-04)

```
DOMContentLoaded → fetch('/now-playing')
  → data.status === "idle" → show #now-playing-idle, hide #now-playing-track
  → data has track → renderCard(data), set currentTrackId = data.track_id
```

### SSE Reconnect Re-Hydration (NOW-05)

```
es.onopen → fetch('/now-playing')  [same logic as page-load]
```

### SSE Event Routing (NOW-02, NOW-03, NOW-07)

```
es.onmessage → parse evt.type
  "track_change" → renderCard(evt), set currentTrackId = evt.track_id, setBadge("evaluating")
  "eval_result"  → if evt.track_id !== currentTrackId: ignore
                   else: updateBadge(evt.eval_state)
  "skip"         → prependSkipItem(evt)  [existing logic, unchanged]
  "five_skip_warning" → show #skip-banner  [existing logic, unchanged]
```

### Skip Button (SKIP-01, SKIP-04)

```
#skip-btn click →
  1. skipBtn.disabled = true  (SKIP-04: prevent double-fire)
  2. skipError.textContent = ''
  3. fetch('POST /skip')
  4a. resp.ok → no UI change (next track_change SSE event handles card update)
  4b. resp.status === 503 → skipError.textContent = 'Skip failed — try again.'
      setTimeout(3000) to clear
  5. skipBtn.disabled = false  (always, in finally)
```

### Module-Level State Variables

```js
let currentTrackId = null;   // track_id of currently displayed card (NOW-07 guard)
```

Added alongside existing `let fsmEnabled = ...` pattern (CONTEXT.md established patterns).

### Animation

Badge state transitions: use `.feed-new` fadeIn animation (`opacity 0 → 1, 0.2s ease-in`) when badge modifier class changes. Apply class, then remove after animation completes. This reuses the existing `@keyframes fadeIn` — no new CSS needed.

## Accessibility

- `aria-live="polite"` on `#now-playing-card` — screen reader announces badge state changes without interrupting (Claude's discretion, CONTEXT.md)
- `aria-label="Skip current track"` on `#skip-btn` — button label is unambiguous when album art is the adjacent visual
- `#now-playing-art` uses `alt=""` — decorative; screen reader skips it
- Skip button `disabled` attribute is set natively — screen readers announce button as unavailable during in-flight request
- `#now-playing-badge` role: no explicit ARIA role needed; text content announces state when `aria-live` region updates
