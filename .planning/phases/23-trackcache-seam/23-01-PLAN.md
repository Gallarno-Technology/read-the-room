## Threat Model (ASVS Level 1)

| Threat | Vector | Mitigation | Severity |
|--------|--------|------------|----------|
| SQL injection via track_id | Caller controls track_id passed to `get()` and `put()` | All SQL uses parameterized queries (`?` placeholders) — never string interpolation | HIGH → BLOCKED |
| Untrusted data written to eval_results | TrackEvalResult fields come from scanner pipeline output | All fields are primitives (str, int, bool); `int()` cast for booleans prevents type confusion; `action` and `reason` are set by internal logic, not user input | LOW |
| DB path traversal | `db_path` arg in `SQLiteTrackCache.__init__` | `db_path` is set at daemon startup from `LYRICS_DB_PATH` env var — the same path already trusted by `LyricsService`; no new attack surface | LOW |
| Stale cache serving wrong result | Cache keyed by `spotify_track_id` | Spotify track IDs are immutable per track; a track's content never changes after release; stale-cache risk is negligible | LOW |

**High-severity threats:** SQL injection is the only high-severity threat. Parameterized queries block it entirely. No additional controls required.
</threat_model>
